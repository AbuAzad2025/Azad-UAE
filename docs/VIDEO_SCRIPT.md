# Video Scripts — Azadexa ERP

## 1. Onboarding Overview (5 minutes)

**Scene:** Screen recording of dashboard.
**Voiceover (Arabic):**

مرحباً بك في أزادكسا ERP. هذا هو لوحة التحكم الرئيسية. هنا ترى إجمالي المبيعات اليوم، المخزون المنخفض، والمهام العاجلة. كل رقم يتحدث لحظياً.

لإنشاء فاتورة: اضغط "مبيعات" ثم "فاتورة جديدة". اختر العميل، أضف المنتجات، واضغط "حفظ". النظام يخصم المخزون تلقائياً وينشئ القيد المحاسبي.

للتحقق من المخزون: اضغط "منتجات". الأحمر يعني إعادة طلب. الأخضر يعني وافر.

**Scene:** POS screen.

نقاط البيع سريعة: امسح الباركود، اضغط "دفع"، استلم النقدية أو البطاقة. انتهى.

**CTA:** احجز عرضاً توضيحياً مجانياً على azadsystems.com.

## 2. Quick Start — First Sale (3 minutes)

**Scene:** Login → Dashboard → Sales → Create.
**Voiceover (Arabic):**

لننشئ أول مبيعة في 60 ثانية.

1. اضغط "مبيعات" ثم "جديد".
2. اختر العميل من القائمة أو أضف عميلاً جديداً.
3. اضغط "إضافة منتج" وامسح الباركود أو اكتب SKU.
4. اضبط الكمية والسعر. النظام يطبق تلقائياً أي عروض نشطة.
5. اضغط "حفظ" ثم "طباعة".

الفاتورة جاهزة: شعارك، بياناتك، ضريبة القيمة المضافة، QR Code.

**Scene:** GL journal entry.

انظر: القيد المحاسبي تلقائي. مدين المبيعات، دائن المخزون.

## 3. Inventory Management Tutorial (5 minutes)

**Scene:** Products list → Warehouse view.
**Voiceover (Arabic):**

إدارة المخزون بثلاث خطوات.

الخطوة الأولى: التتبع. كل منتج له SKU فريد. يمكن تتبع التسلسلي لكل قطعة فردية.

الخطوة الثانية: التكلفة. النظام يحسب متوسط التكلفة المرجح (WAC) تلقائياً عند كل عملية شراء.

الخطوة الثالثة: التسوية. في نهاية كل شهر، اضغط "تسوية مخزونية". قارن العدد الفعلي مع النظام. أي فرق ينشئ قيد تعديل.

**Scene:** Stock transfer.

لنقل المخزون بين المستودعات: اختر "نقل"، المصدر، الوجهة، المنتجات، الكميات. وصلت.

## 4. POS Checkout Tutorial (4 minutes)

**Scene:** POS screen → Scan → Payment → Receipt.
**Voiceover (Arabic):**

نقطة البيع مصممة للسرعة.

1. امسح الباركود. المنتج يظهر فوراً مع السعر والضريبة.
2. أضف منتجات أخرى. النظام يحسب الإجمالي.
3. اضغط "العروض" لتطبيق خصم أو BOGO.
4. اضغط "دفع". اختر: نقدي، بطاقة، شيك، أو دمج.
5. استلم المبلغ. النظام يطبع الإيصال ويفتح الدرج.

**Scene:** Manager override.

للإرجاع: اضغط "إرجاع"، أدخل رقم الفاتورة الأصلية، اختر المنتجات. يتطلب تأكيد المشرف.

## 5. AI Assistant Tutorial (3 minutes)

**Scene:** Chat interface.
**Voiceover (Arabic):**

المساعد الذكي يتعلم من بيانات عملك.

اسأله: "ما المنتجات التي يجب إعادة طلبها؟"
يجيب: قائمة بالمنتجات مع الكميات المقترحة بناءً على سرعة البيع.

اسأله: "ما توصية السعر لهذا المنتج؟"
يجيب: السعر الأمثل بناءً على التكلفة، المنافسة، والطلب.

اسأله: "كيف أداء فرع دبي هذا الشهر؟"
يجيب: إجمالي المبيعات، صافي الربح، ومقارنة مع الشهر السابق.

كل إجابة مبنية على بياناتك الحقيقية، وليس على الإنترنت العام.

## 6. Advanced — GL Reconciliation (7 minutes)

**Scene:** Ledger → Trial Balance → Reconciliation.
**Voiceover (Arabic):**

للمحاسبين: كيفية التسوية الشهرية.

1. اضغط "دفتر الأستاذ" ثم "ميزان المراجعة".
2. تأكد من أن إجمالي المدين يساوي إجمالي الدائن.
3. اضغط "تسوية بنكية". استورد كشف الحساب.
4. طابق المعاملات تلقائياً. الغريب يظهر بالأحمر.
5. للغريب: حدده، أدخل القيد التصحيحي، واضغط "تسوية".

**Scene:** Month-end.

نهاية الشهر: اضغط "إقفال" لقفل الفترة. لا يمكن تعديل القيود بعد الإقفال.
