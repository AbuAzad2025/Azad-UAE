## 3. Code of Conduct | قواعد السلوك

**EN:**
3.1 Treat colleagues with respect. No harassment, discrimination, or retaliation.
3.2 Protect company and customer data. Do not share credentials, screenshots, or database dumps externally.
3.3 Report conflicts of interest (e.g., side projects in the same industry).
3.4 Do not use company resources for personal gain.
3.5 Attend team meetings. Camera-on for video calls.

**AR:**
3.1 عامل الزملاء باحترام. لا تحرش أو تمييز أو انتقام.
3.2 احمِ بيانات الشركة والعملاء. لا تشارك بيانات الاعتماد أو لقطات الشاشة أو dumps قاعدة البيانات خارجياً.
3.3 أبلغ عن تضارب المصالح (مثل المشاريع الجانبية في نفس الصناعة).
3.4 لا تستخدم موارد الشركة لتحقيق مكاسب شخصية.
3.5 حضور اجتماعات الفريق. الكاميرا مفتوحة لمكالمات الفيديو.

## 4. Security and Confidentiality | الأمان والسرية

**EN:**
4.1 Use a password manager. All accounts must have unique, strong passwords.
4.2 Enable 2FA on GitHub, Slack, and all production access points.
4.3 Lock your computer when away.
4.4 Do not install unapproved software on work devices.
4.5 Customer data is confidential. Do not access tenant data without business justification.

**AR:**
4.1 استخدم مدير كلمات مرور. يجب أن يكون لجميع الحسابات كلمات مرور فريدة وقوية.
4.2 فعّل 2FA على GitHub و Slack وجميع نقاط الوصول الإنتاجية.
4.3 أغلق حاسوبك عند الابتعاد.
4.4 لا تثبت برمجيات غير مُعتمدة على أجهزة العمل.
4.5 بيانات العملاء سرية. لا تصل إلى بيانات المستأجر دون مبرر تجاري.

## 5. Development Standards | معايير التطوير

**EN:**
5.1 All code must pass: tests, lint (ruff), type check (mypy), and GRIMOIRE compliance.
5.2 No direct `commit()` or `rollback()` in services.
5.3 Every route change needs a corresponding test.
5.4 Commit messages follow conventional commit style.
5.5 PRs (if used) require at least one review before merge.

**AR:**
5.1 يجب أن يمر كل كود بـ: الاختبارات، lint (ruff)، فحص النوع (mypy)، والامتثال لـ GRIMOIRE.
5.2 لا `commit()` أو `rollback()` مباشرة في الخدمات.
5.3 كل تغيير في المسارات يحتاج اختباراً مقابلاً.
5.4 رسائل الالتزام تتبع أسلوب الالتزام التقليدي.
5.5 PRs (إن استُخدمت) تتطلب مراجعة واحدة على الأقل قبل الدمج.
