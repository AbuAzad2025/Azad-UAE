# Acceptable Use Policy — AZAD Intelligent Systems

## 1. Scope

This policy applies to all employees, contractors, and authorized users of AZAD Intelligent Systems resources, including computers, networks, cloud services, and customer data access.

## 2. Authorized Use

| Resource | Permitted Use |
|----------|---------------|
| Company laptops | Work-related tasks, approved learning |
| Internet | Research, communication, cloud services |
| Email | Business correspondence |
| GitHub | Code contribution, issue tracking |
| Slack | Team communication |
| Azadexa ERP (internal tenant) | Testing, demos, training |

## 3. Prohibited Use

3.1 Illegal Activities
- No hacking, fraud, piracy, or distribution of illegal content.
- No use of company resources for cryptocurrency mining.

3.2 Security Risks
- No sharing of passwords or API keys.
- No bypassing security controls (VPN, firewalls, RBAC).
- No installing unauthorized software or browser extensions.
- No connecting unknown USB devices to company hardware.

3.3 Data Mishandling
- No copying customer databases to personal devices.
- No emailing tenant data to personal accounts.
- No posting screenshots containing customer data on social media.
- No using customer lists for personal business.

3.4 Network Abuse
- No excessive streaming, gaming, or large file downloads unrelated to work.
- No running port scanners or penetration testing tools without authorization.

3.5 Harassment
- No sending offensive, discriminatory, or threatening messages.
- No using company communication channels for personal disputes.

## 4. Monitoring

AZAD reserves the right to monitor:
- Network traffic.
- Email and Slack communications for compliance and security.
- Access logs to production systems.

Monitoring is for security, compliance, and operational integrity — not for micromanagement.

## 5. BYOD Policy

| Aspect | Rule |
|--------|------|
| Eligibility | Approved on case-by-case basis |
| Security | Device must have screen lock, encryption, and MDM agent |
| Access | VPN required for all internal resource access |
| Separation | Work profile/container must be used |
| Exit | Work data is remotely wiped upon termination |

## 6. Social Media

| Rule | Detail |
|------|--------|
| Official accounts | Managed by Marketing; no personal posting on behalf of AZAD |
| Personal accounts | Employees may share company news with approval |
| Confidentiality | No disclosure of unreleased features, customer names, or revenue |
| Opinions | Personal opinions must not appear as company positions |

## 7. Consequences

| Violation | Consequence |
|-----------|-------------|
| Minor (first offense) | Verbal warning + retraining |
| Moderate (repeat or serious) | Written warning + restricted access |
| Major (data breach, fraud, illegal) | Immediate termination + legal action |

## 8. Policy Review

Reviewed annually. Effective immediately upon publication.
