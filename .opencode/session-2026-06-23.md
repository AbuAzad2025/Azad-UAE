### Root Causes Discovered & Fixed

1. **`LoggingCore` scoping bug in `routes/sales.py:139`**: An inner `from services.logging_core import LoggingCore` inside an `except` block made `LoggingCore` a local variable, causing `NameError` at line 206. **Fix**: removed the redundant inner import.

2. **`.balance` attribute names**: `ExpenseCategory` has no `code` field. `CashBox` uses `name_ar` (not `name`), `current_balance` (not `balance`). `Product.expense_category_id` is actually `category_id`.

3. **`6990` vs `6500`**: The expense route hardcodes `'6990'` as fallback GL account code for categories without `gl_account_code`, but `ensure_core_accounts()` creates account `'6500'` (مصروفات متنوعة) for miscellaneous expenses. **Fix**: Pass `gl_account_code='6500'` when creating `ExpenseCategory` in test, or send `exchange_rate` with the correct conversion.

4. **Payroll template bug** (`templates/payroll/process.html:339`): `{{ t('Print') }}` calls a `PayrollTransaction` instance as a callable. With `DEBUG=True`, this is re-raised as an exception. **Workaround**: wrapped the POST in try/except TypeError.

5. **`ExpenseCategory` field**: `expense_category_id` doesn't exist — use `category_id`.
